# Explanation

Explanation helps readers build mental models.

## Goals

- Prose descriptions of how things work and why they work that way
- Avoid showing too much code and focus on concepts and architecture

## Audience

- Developers
- Mid to advanced users

## Content

The explanation space is organized around concepts like core services, GUI behavior, and devices rather than around repository names.

This is where the documentation should answer questions like:

- why the system is organized this way
- how core services, widgets, and device layers relate to one another
- what assumptions shape the development model
- where responsibilities begin and end across the runtime stack
